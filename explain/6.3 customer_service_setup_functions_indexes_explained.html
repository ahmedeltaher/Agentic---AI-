🤖 إعداد نموذج اللغة الذكية (LLM)

استخدام Azure OpenAI

المدرس اختار Azure OpenAI كنموذج اللغة الأساسي، لكن وضح إنك ممكن تستخدم أي نموذج تاني بس لازم تغير الإعدادات والوظائف المناسبة في LlamaIndex.

المطلوب للإعداد

🔑 البيانات الأساسية

  • API Key: مفتاح الوصول لحسابك
  • Azure Endpoint: عنوان الخدمة على Azure
  • API Version: إصدار الـ API المستخدم

مهم جداً: لازم تستبدل القيم دي بالقيم من حسابك الشخصي!

⚠️ تحذير مهم - اختيار النموذج

المدرس نبه إن GPT-4 بس هو اللي بيدعم Function Calling في الـ API version المستخدم، GPT-3.5 مش هيشتغل على Azure للمهمة دي!

إعداد نموذج التضمين (Embedding)

بالإضافة للـ LLM، هنحتاج نموذج embedding عشان نقدر نعالج مستند الدعم الفني ونحوله لفهرس قابل للبحث.

💡 ليه نحتاج Embedding Model؟

النموذج ده مسؤول عن تحويل النصوص لأرقام (vectors) عشان الكمبيوتر يقدر يفهمها ويبحث فيها بسرعة ودقة.

⚙️ إنشاء الوظائف الثلاثة

الوظائف المطلوبة

هننشئ الثلاث وظائف اللي اتكلمنا عنهم في التصميم، وكل وظيفة هتستخدم قوائم مبرمجة مسبقاً لأغراض التوضيح.

🏢 ملاحظة للتطبيق الحقيقي

في التطبيقات الحقيقية، الوظائف دي مش هتستخدم قوائم ثابتة، بل هتتصل بقواعد البيانات الفعلية للشركة عشان تجيب المعلومات الحديثة والدقيقة.
