الهدف من الفيديو

دلوقتي جه الوقت إننا نجمع كل الحاجات اللي عملناها في الفيديو اللي فات ونبني الـ Customer Service AI Agent الكامل! هنبدأ من الوظائف والـ query engine اللي جهزناهم ونحولهم لـ agent شغال.

المدرس بيوضح إن المرحلة دي هتتكون من خطوات محددة:

  • تحويل الوظائف لأدوات: استخدام Function Tool Interface
  • إنشاء Query Engine Tool: للبحث في المستندات
  • بناء Agent Worker: محرك تنفيذ المهام
  • إنشاء Agent Runner: الواجهة النهائية للـ Agent

💡 ترابط المراحل

الفيديو ده بيبني على كل اللي عملناه قبل كده - الوظائف الثلاثة ومعالجة المستند - ويجمعهم في نظام واحد متكامل.

🛠️ إنشاء الأدوات من الوظائف

استخدام Function Tool Interface

المدرس بيستخدم Function Tool Interface الموجود في LlamaIndex عشان يحول كل وظيفة من الوظائف الثلاثة لأداة قابلة للاستخدام في الـ Agent.

الأدوات الثلاثة الأساسية

🛒 Order Items Tool

مبنية على: get_order_items() function

الوظيفة: تجيب قائمة المنتجات لأي رقم طلب

طريقة التحويل: تغليف الـ function في Function Tool Interface

🚚 Delivery Date Tool

مبنية على: get_delivery_date() function

الوظيفة: ترجع تاريخ التسليم المتوقع

طريقة التحويل: استخدام نفس الـ Interface لتحويلها لأداة

↩️ Return Policy Tool

مبنية على: get_item_return_days() function

الوظيفة: تحدد فترة الإرجاع المسموحة

طريقة التحويل: تطبيق نفس النمط لإنشاء الأداة

الأداة الرابعة: Query Engine Tool

📚 Support Query Engine Tool

مبنية على: الـ query engine اللي عملناه للـ customer_service.pdf

الوظيفة: البحث في مستندات الدعم الفني

طريقة الإنشاء: تحويل الـ query engine لأداة قابلة للاستخدام

🔍 ليه نحتاج تحويل لأدوات؟

الـ Agent مش بيتعامل مع الـ functions مباشرة، بل محتاج يفهم إزاي يستدعي كل وظيفة ومتى يستخدمها. الـ Tools بتوفر له الواجهة المناسبة لده.

⚙️ بناء Agent Worker

فهم مفهوم Agent Worker

المدرس بيوضح إن الـ Agent Worker في LlamaIndex بيكافئ الـ Task Executor اللي اتكلمنا عنه في مفاهيم الـ Agentic AI.

مكونات الـ Agent Worker

📋 المطلوب لإنشاء Agent Worker

  • قائمة الأدوات: الأربع أدوات اللي جهزناهم
  • نموذج اللغة (LLM): GPT-4 اللي جهزناه قبل كده
  • إعدادات إضافية: زي مستوى التفصيل (verbose)

👁️ إعداد Verbose Level

المدرس حط الـ verbose level على true عشان نقدر نشوف إزاي الـ Agent بيفكر ويتخذ قراراته داخلياً. ده مفيد جداً للفهم والـ debugging.

فائدته: نشوف كل خطوة بيعملها الـ Agent وإزاي بيختار الأداة المناسبة لكل سؤال.

🧠 دور الـ Agent Worker

الـ Agent Worker هو المسؤول عن:

  • فهم السؤال أو الطلب من العميل
  • تحديد الأداة أو الأدوات المناسبة
  • تنفيذ المهمة باستخدام الأدوات المختارة
  • تجميع النتائج وصياغة الإجابة النهائية

🏃‍♂️ إنشاء Agent Runner

الخطوة الأخيرة: Agent Runner

بعد ما عملنا الـ Agent Worker، المدرس بيستخدم Agent Runner Interface في LlamaIndex عشان ينشئ الواجهة النهائية للـ Agent.

دور الـ Agent Runner

🎯 وظيفة Agent Runner

الـ Agent Runner بيمثل الواجهة الخارجية اللي المستخدمين هيتعاملوا معاها. هو بياخد الـ Agent Worker كمعطى ويوفر طرق سهلة للتفاعل مع الـ Agent.

العلاقة بين المكونات

Agent Runner ← Agent Worker ← Tools ← Functions

كل طبقة بتبني على اللي قبلها وتضيف لها إمكانيات جديدة

🔄 تدفق العمل

المستخدم يتكلم مع الـ Agent Runner، اللي بيمرر الطلب للـ Agent Worker، اللي بدوره يستخدم الأدوات المناسبة عشان يجيب المعلومات ويرجع الإجابة.

🔍 فهم آلية العمل الداخلية

🧠 كيف يفكر الـ Agent؟

لما العميل يسأل سؤال، الـ Agent بيعدي بالخطوات دي:

  1. فهم السؤال: تحليل النص وفهم المطلوب
  2. تحديد الأدوات: اختيار الأداة أو الأدوات المناسبة
  3. تنفيذ المهمة: استدعاء الأدوات بالمعطيات الصحيحة
  4. تجميع النتائج: ربط المعلومات من أدوات مختلفة لو لزم الأمر
  5. صياغة الإجابة: تقديم رد واضح ومفيد للعميل

👀 فوائد الـ Verbose Mode

مع تفعيل الـ verbose، هنقدر نشوف:

  • إيه الأداة اللي الـ Agent اختارها
  • إيه المعطيات اللي مررها للأداة
  • إيه النتيجة اللي رجعت من كل أداة
  • إزاي جمع المعلومات وكون الإجابة النهائية

🎯 أمثلة على سيناريوهات العمل

📞 سيناريو: سؤال بسيط

العميل: "إيه اللي في طلبي رقم 123؟"

الـ Agent: هيستخدم Order Items Tool بس

النتيجة: قائمة بكل منتجات الطلب

🔍 سيناريو: سؤال معقد

العميل: "طلبي رقم 456 فيه إيه وامتى هيوصل وممكن أرجع حاجة منه؟"

الـ Agent: هيستخدم تلات أدوات: Order Items + Delivery Date + Return Policy

النتيجة: إجابة شاملة تغطي كل الأسئلة

❓ سيناريو: سؤال عام

العميل: "إيه سياسة الشحن في الشركة؟"

الـ Agent: هيستخدم Support Query Engine Tool

النتيجة: معلومات من مستند customer_service.pdf

🔮 الخطوة القادمة

🎬 الفيديو القادم: التجربة العملية

المدرس وضح إن الفيديو الجاي هيركز على تجربة الـ Agent مع أهداف وأسئلة مختلفة عشان نشوف إزاي بيشتغل في الواقع.

🎯 ما نتوقعه في الفيديو القادم

  • تجربة أسئلة مختلفة على الـ Agent
  • مراقبة كيفية اختيار الأدوات
  • تحليل جودة ودقة الإجابات
  • استكشاف حالات معقدة ومتعددة الأدوات

💡 نصائح للمطورين

🛠️ أفضل الممارسات

  • اختبار تدريجي: اختبر كل أداة لوحدها قبل التجميع
  • استخدام Verbose: فعل الـ verbose mode أثناء التطوير
  • معالجة الأخطاء: حط error handling مناسب لكل أداة
  • مراقبة الأداء: تتبع سرعة ودقة كل أداة

🔍 نصائح التصحيح

  • لو الـ Agent مش بيختار الأداة الصح، راجع وصف الأدوات
  • لو النتائج مش دقيقة، اختبر الـ functions الأساسية
  • لو Query Engine مش بيرجع نتائج كويسة، راجع الفهرس
  • استخدم الـ verbose logs عشان تفهم مسار التفكير
